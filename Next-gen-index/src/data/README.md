# 区域数据文件说明

## 文件结构

- `regions.ts`: 包含所有国家的行政区域层级数据和坐标数据

## 数据结构

### REGION_HIERARCHY
三级层级结构：`国家 -> 省/州 -> 市/区`

```typescript
{
  "国家名称": {
    "省/州名称": ["市/区1", "市/区2", ...],
    ...
  },
  ...
}
```

### REGION_CENTERS
区域中心点坐标数据，用于地图定位和边界生成

```typescript
{
  "国家名称": {
    "省/州名称": {
      "市/区名称": { lat: 纬度, lng: 经度 },
      ...
    },
    ...
  },
  ...
}
```

## 如何添加新区域

### 1. 添加新的国家

在 `REGION_HIERARCHY` 和 `REGION_CENTERS` 中添加新国家：

```typescript
export const REGION_HIERARCHY = {
  // ... 现有国家
  "新国家名称": {
    "省/州名称": ["市/区1", "市/区2"],
    ...
  }
};

export const REGION_CENTERS = {
  // ... 现有国家
  "新国家名称": {
    "省/州名称": {
      "市/区名称": { lat: 纬度, lng: 经度 },
      ...
    },
    ...
  }
};
```

### 2. 添加现有国家的新省/州

在对应国家的对象中添加新的省/州：

```typescript
"China": {
  // ... 现有省份
  "新省份": ["城市1", "城市2", ...]
}
```

### 3. 添加现有省/州的新市/区

在对应省/州数组中添加新城市：

```typescript
"Beijing": [
  // ... 现有城市
  "新城市"
]
```

## 坐标数据获取

坐标数据可以从以下来源获取：
- Google Maps Geocoding API
- OpenStreetMap Nominatim API
- GeoNames 数据库
- 官方统计机构数据

## 注意事项

1. **数据一致性**: 确保 `REGION_HIERARCHY` 和 `REGION_CENTERS` 中的区域名称完全一致
2. **坐标精度**: 使用主要城市的中心点坐标，精度到小数点后4位即可
3. **命名规范**: 
   - 国家名称使用英文标准名称
   - 省/州名称使用英文或当地语言的标准名称
   - 市/区名称使用当地语言的标准名称
4. **数据更新**: 定期检查并更新区域数据，确保与实际情况一致
