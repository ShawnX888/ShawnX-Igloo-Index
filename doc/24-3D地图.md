# 24-3D地图

## 步骤概述

### 步骤编号和名称
**步骤24**：3D地图功能实现

### 步骤目标

**实施目标**：实现 2D 和 Vector Map + 3D Buildings 两种模式的切换功能
- **2D 模式**：传统平面地图视图
- **Vector Map + 3D Buildings**：矢量地图 3D 建筑（白模风格）

支持用户在地图的两种模式之间自由切换，通过统一的样式配置管理不同模式下的图层显示效果，提供流畅的切换动画和一致的用户体验。

### 关于 Photorealistic 3D Maps 的说明

**决策**：放弃 Photorealistic 3D Maps 模式的实现

**弃用原因**：

1. **架构不兼容**
   - 从实现架构的角度来看，普通 Google Map (Vector) 与 Photorealistic 3D Map 并不是同一个实例的两种"模式"
   - 它们是两个完全独立的渲染引擎（`Map` vs `Map3DElement`）
   - 实现三种模式切换需要管理两个不同的地图实例，架构复杂度高
   - 图层在不同实例间迁移需要额外的适配工作

2. **应用场景不匹配**
   - Photorealistic 3D Maps 适合沉浸式展示、旅游导览、房产周边环境查看等场景
   - 本项目是数据可视化、风险分析场景，需要清晰展示数据图层
   - Photorealistic 3D Maps 的真实照片纹理可能会干扰数据图层的可见性
   - Vector Map + 3D Buildings 的"白模"风格更适合数据可视化场景

3. **实施复杂度高**
   - 需要维护两套地图实例（Map 和 Map3DElement）
   - 图层 Hook 需要适配两种不同的地图类型
   - 状态同步和切换逻辑复杂
   - 性能开销大（高阶计费）

**结论**：专注于实现 2D ↔ Vector Map + 3D Buildings 的切换，满足数据可视化和风险分析的核心需求。

## 3D 地图模式说明

### 3D 模式选择

**当前实施方案**：Vector Map + 3D Buildings（矢量地图 3D 建筑）

**技术方案**：
- 使用标准 `maps` 库和 `Map` 类
- 通过 `tilt` 和 `heading` 参数实现 3D 视角
- **"白模"风格**：抽象的矢量方块，侧重于展示建筑轮廓和高度，表面无纹理
- 适合数据可视化、风险分析等场景

**为什么选择 Vector Map + 3D Buildings**：
1. 与现有架构兼容：使用同一个 `Map` 实例，只需调整参数即可切换 2D/3D
2. 数据可视化友好：白模风格不会干扰数据图层的可见性
3. 实现简单：无需管理多个地图实例
4. 性能可控：标准计费，成本可控
5. 支持样式自定义：便于调优数据图层的显示效果

### 关于 Photorealistic 3D Maps 的说明

**状态**：已放弃实现

**原因**：
1. **架构不兼容**：Photorealistic 3D Maps 使用 `Map3DElement` 类，与 `Map` 类是完全独立的渲染引擎，不是同一个实例的两种模式
2. **应用场景不匹配**：适合沉浸式展示、旅游导览，不适合数据可视化场景
3. **实施复杂度高**：需要管理两个不同的地图实例，图层迁移复杂

**参考信息**（仅用于了解，不实施）：
- 使用 `maps3d` 库和 `Map3DElement` 类

## 核心设计

### 1. 样式配置统一管理

**设计目标**：通过统一的配置文件管理不同 `mapMode` 下的样式，便于后续调优和维护。

**实现方案**：
- 创建 `mapModeStyles.ts` 配置文件
- 定义 2D 和 Vector Map + 3D Buildings 模式下的样式配置对象
- 各图层 Hook 从配置文件读取样式
- 支持后续扩展和调优

**配置文件结构**：
```typescript
// 2D 和 Vector Map + 3D Buildings
export const mapModeStyles = {
  '2d': {
    boundary: { strokeWeight: 1, strokeOpacity: 0.4, ... },
    heatmap: { fillOpacity: { min: 0.15, max: 0.7 }, ... },
    markers: { size: 1.0, shadow: 'light' },
    map: { zoom: 11, tilt: 0, rotateControl: false }
  },
  '3d': {
    // Vector Map + 3D Buildings
    boundary: { strokeWeight: 3, strokeOpacity: 0.8, ... },
    heatmap: { fillOpacity: { min: 0.2, max: 0.8 }, ... },
    markers: { size: 1.2, shadow: 'enhanced' },
    map: { zoom: 18, tilt: 45, heading: 0, rotateControl: true }
  }
};
```

### 2. 切换动画和过渡效果

**设计目标**：提供流畅的视角切换体验，避免突兀的视角变化。

**实现方案**：
- 使用 Google Maps 的 `setTilt()` 和 `setZoom()` 方法（支持平滑过渡）
- 添加过渡状态管理（`isTransitioning`）
- 在切换过程中禁用用户交互
- 设置合理的过渡时长（建议 500-800ms）

**动画流程**：
1. 用户点击切换按钮
2. 设置 `isTransitioning = true`，禁用交互
3. 平滑过渡地图参数：
   - 2D → Vector 3D：使用 `setTilt()`、`setZoom()`、`setHeading()`
   - Vector 3D → 2D：使用 `setTilt()`、`setZoom()`、`setHeading()`
4. 过渡完成后，设置 `isTransitioning = false`
5. 更新图层样式（从配置文件读取）

### 3. 图层兼容性

**所有图层完全兼容**：
- ✅ 区域边界图层（Data Layer）：自动适配 3D 视角
- ✅ 降雨量热力图图层（Data Layer）：正常显示

### 2. 模式切换动画实现

```typescript
// 伪代码：切换逻辑（2D ↔ Vector Map + 3D Buildings）
useEffect(() => {
  if (!mapInstanceRef.current || !mapsLoaded) return;
  
  const map = mapInstanceRef.current;
  const targetMode = mapMode;
  const styles = mapModeStyles[targetMode];
  
  // 1. 设置过渡状态
  setIsTransitioning(true);
  
  // 2. 平滑过渡地图参数
  const mapConfig = styles.map as Vector3DMapConfigStyles;
  map.setTilt(mapConfig.tilt);
  map.setHeading(mapConfig.heading);
  if (map.getZoom() < mapConfig.zoom) {
    map.setZoom(mapConfig.zoom);
  }
  map.setOptions({ rotateControl: mapConfig.rotateControl });
  
  // 3. 等待过渡完成（Google Maps 自动处理）
  setTimeout(() => {
    setIsTransitioning(false);
  }, 800); // 过渡时长
}, [mapMode, mapsLoaded]);
```

- 3D 模式渲染开销更高，注意低端设备性能
- 切换动画时长建议 500-800ms，避免过长影响体验
- 考虑在低性能设备上禁用 3D 模式或降低倾斜角度

### 3. 用户体验

- 切换时自动调整缩放级别，确保 3D 效果可见
- 过渡期间禁用用户交互，避免操作冲突
- 在 3D 模式下启用旋转控制，允许用户自由调整视角
- 提供清晰的视觉反馈（按钮状态、加载提示）

### 4. 样式调优

- 所有样式参数集中在配置文件中，便于后续调优
- 建议先使用默认配置，根据实际效果逐步调整
- 保持 2D 和 3D 模式的视觉一致性

### 5. 兼容性

- 所有图层（Data Layer、Advanced Markers）在 3D 模式下完全兼容
- 样式配置系统支持后续扩展（如添加新的模式或图层类型）
- 保持向后兼容，默认使用 2D 模式
