# v2 页面设计提案（数据大屏 / 科技感态势感知）

> 阶段：头脑风暴（不考虑后端与数据可得性）  
> 目标：在 **地图 + 数据面板** 的联动框架下，清晰呈现「省级全局保单/理赔排名」与「所选区域理赔趋势&风险关联」，同时让页面更“像一张会呼吸的指挥中心”。

---

## 0. 设计北极星（North Star）

用户打开页面后 10 秒内应做到：

- **一眼看到态势**：当前省份的保单与理赔分布（热力/分级）+ Top 排名，知道“哪里最活跃 / 哪里风险最高 / 哪里赔得最多”。
- **一键下钻到细节**：点击任意区域，右侧立即出现该区域的理赔概览、趋势、与风险事件的时间对齐关联。
- **可解释性强**：所有关键数字都能回答“为什么”（有对比、有时间轴、有阈值/事件标记）。

---

## 1. 信息分层（让内容层次分明）

把信息拆成 3 个层级（强制分区，避免“所有东西都堆在同一张图里”）：

- **L0 省级态势（全局）**：保单 / 理赔 KPI + 排名 + 省级趋势  
  - 回答：**这一个省整体如何？排名如何？变化方向如何？**
- **L1 区域态势（选中区域）**：理赔概览 + 理赔趋势 + 风险事件 + 关键相关性  
  - 回答：**这个区为什么突出？什么时候开始异常？与风险是否相关？**
- **L2 证据链（解释与溯源）**：可点击的明细列表/事件时间线/对比面板  
  - 回答：**给我证据：哪些理赔、对应哪些风险、触发了什么规则？**

将“时间”作为横向维度贯穿 L0/L1/L2：**同一条时间轴，不同层级同步刷选**。

---

## 2. 推荐主布局：Mission Control 三联屏（科技感最强、结构最稳）

核心：**中间地图是空间过滤器**，左右是“上下文 + 细节”，底部是“时间轴总控”。

### 2.1 页面骨架（信息架构草图）

```
┌────────────────────────────────────────────────────────────────────────────┐
│ 顶栏：品牌/环境 | 全局时间范围 | 数据类型(历史/预测) | 图层(风险/保单/理赔) │
└────────────────────────────────────────────────────────────────────────────┘
┌───────────────┬───────────────────────────────────────┬───────────────────┐
│ L0 省级态势     │ 地图（主舞台）                         │ L1 区域态势         │
│                │                                       │                   │
│ [省级KPI]       │ ① 分级底图：保单密度/理赔密度/风险等级  │ [区域理赔KPI]       │
│ [保单排名 Top]  │ ② 事件：风险点/理赔点（可切换）         │ [理赔趋势 vs 风险]  │
│ [理赔排名 Top]  │ ③ 选区：Hover 高亮 / Click 锁定         │ [关联分析：散点/矩阵]│
│ [省级趋势]      │ ④ Legend：颜色/阈值/等级说明             │ [证据链列表]         │
└───────────────┴───────────────────────────────────────┴───────────────────┘
┌────────────────────────────────────────────────────────────────────────────┐
│ 时间轴总控（Brush + 播放）：刷选后 L0/L1 全部联动更新                         │
└────────────────────────────────────────────────────────────────────────────┘
```

### 2.2 交互主线（让“联动”清晰可预期）

- **地图 Hover（轻交互）**：仅高亮区域边界 + 弹出 mini tooltip（3 个数：保单、理赔、风险）。不改变右侧详情。
- **地图 Click（重交互/锁定）**：锁定选区（L1 面板刷新为该区详情）；L0 面板自动滚动/高亮该区在排名中的位置。
- **排名 Click（反向驱动地图）**：点击 Top 排名中的区 → 地图飞到该区并锁定（强联动，符合“排行榜就是导航”）。
- **时间轴 Brush（全局过滤）**：刷选一个时间窗口 → L0 省级趋势、L1 理赔趋势、风险事件标记同步缩放到同一时间范围。
- **Pin/Compare（对比模式）**：支持“钉住一个区域”作为基线，再点击另一个区域做对比（对比只在 L1 的趋势与 KPI 生效，避免地图混乱）。

## 3. 三类新增信息如何“放得漂亮”

对应你提出的 3 个需求，我建议这样组织视觉与交互：

### 3.1 省级保单统计与排名（L0 左侧）

- **省级 KPI 行**（3–5 个）：总保单数、活跃保单、覆盖金额（或保额）、近 7/30 天新增、环比/同比
- **排名可视化**：**横向条形图（Leaderboard）** + 数值标签  
  - 交互：Hover 显示区名/值/占比；Click 切换地图选区；支持 Sort（按保单数/保费/覆盖等）
- **省级趋势**：折线/面积图（可切换指标：保单新增 vs 活跃 vs 保费）

> 设计语言：左侧像“态势播报台”——数字大、结构紧、层级强。

### 3.2 省级理赔统计与排名（L0 左侧）

与保单一致的结构，但强调“异常与告警”：

- KPI：理赔件数、理赔金额、平均结案时长、拒赔率/通过率、理赔率（理赔件数/保单数）
- 排名：Top 理赔区（件数/金额双维可切换）
- 趋势：理赔件数 vs 金额（可双轴，但要明确图例与轴标签）

> 视觉强调：理赔相关用“告警色系”（红/橙）但**不靠颜色单独表达**，始终配 icon 与文字（可访问性规则）。

### 3.3 所选区域理赔详情、趋势、与 risk 关联（L1 右侧）

右侧做成“证据链面板”，遵循从上到下：**概览 → 趋势 → 关联 → 明细**。

- **概览 KPI**：理赔件数、理赔金额、平均理赔金额、理赔率、最高单笔/最高等级风险对应理赔
- **趋势图（核心）**：同一时间轴对齐显示  
  - 理赔金额/件数（线或面积）  
  - 风险事件（点/竖线标记，按 tier 着色）  
  - 可选：产品阈值带（让“触发-理赔”一眼可解释）
- **关联分析（更吸引眼球）**：  
  - **散点图**：x=风险强度/降雨指标，y=理赔金额；点颜色=风险 tier；点大小=理赔金额或件数  
  - 或 **矩阵热力**：风险等级 × 理赔状态/产品类型（快速看结构性差异）
- **证据链列表**（可折叠）：按时间倒序列出“理赔记录 ↔ 风险事件 ↔ 产品规则提示”

## 4. 科技感视觉系统（不土、不花、但很抓人）

基于 UI/UX 素材库（Dark Mode OLED + Glassmorphism + Financial Dashboard + Bento Grid）的组合建议：

### 4.1 色彩（推荐：深色底 + 可信蓝 + 霓虹点缀）

- **背景**：#0F172A（深海军蓝，稳定且专业）
- **文字**：#F8FAFC（高对比白）
- **边框**：#334155（低调分隔）
- **主色（可信/交互）**：#3B82F6（蓝）
- **强调 CTA**：#8B5CF6（紫）或 #F59E0B（琥珀）
- **风险/告警**：tier1=amber、tier2=orange、tier3=red（并配 icon/标签）

关键规则：

- **卡片是玻璃层**：背景半透明 + 10–20px blur + 细边框（“雾化玻璃”），而不是厚重阴影。
- **可读性优先**：正文/数字对比度至少 **4.5:1**，灰字不要压到看不清。

### 4.2 字体（更“仪表盘”）

两套可选：

- **Tech Startup（更现代、更科技）**：Space Grotesk（标题） + DM Sans（正文）
- **Dashboard Data（更工程、更数据）**：Fira Code（数字/标签） + Fira Sans（正文）

建议：数字与关键指标用等宽或半等宽风格，瞬间“数据感拉满”。

### 4.3 组件质感（让页面“像系统”）

- **Bento Grid**：卡片大小不完全等高，做出“拼贴式大屏”，更像指挥中心。
- **数字动效**：KPI 上使用轻量 count-up（200–400ms），配“涨跌箭头/Delta”。
- **微光效**：只在“当前选中/告警/可交互元素”出现轻微 glow，避免全屏霓虹导致廉价。
- **地图舞台**：地图上保留干净的舞台边缘（8–12px 内边距）+ 角落 HUD 控件（图层、图例、定位、模式）。

### 4.4 动效节奏（吸引眼球但不打扰）

- **Hover 高亮**：边界描边 + 轻微外发光（不改变布局）
- **Click 锁定**：地图飞行（800–1200ms）+ 右侧面板卡片“从右滑入 + 淡入”
- **时间轴刷选**：图表区域“缩放动画” + Loading skeleton（避免突变）
- 尊重 **prefers-reduced-motion**（降低眩晕风险）

## 5. 三套可替换布局（在同一设计语言下的不同取舍）

### 方案 A：Mission Control 三联屏（主推）

```
L0 省级态势（左） | 地图主舞台（中） | L1 区域证据链（右）
底部：全局时间轴（刷选/播放）
```

- **优点**：层级最清晰、科技感最强、联动逻辑最直观  
- **适用**：大屏/桌面端为主，强调“态势感知”

### 方案 B：右侧 Dock + 底部 Storyline（更偏叙事）

```
地图（中/大） + 右侧可伸缩 Dock（省级/区域 Tab）
底部：Storyline（按时间组织：风险 → 理赔 → 产品解释）
```

- **优点**：更适合“讲清楚一件事”，可做演示/路演
- **代价**：省级态势不如三联屏一眼可见

### 方案 C：Bento Intelligence Wall（更吸睛、更“海报化”）

```
地图缩为“参考地图” + 大量 Bento 卡片（KPI/排名/关联/趋势）
卡片之间联动高亮（hover/brush/click）
```

- **优点**：视觉冲击强，非常像大屏
- **代价**：地图的主导性下降，需要更强的卡片组织能力

## 6. 参考网站（找灵感用）

这些都属于“地图 + 多图层 + 时间/数据联动”的强参考，适合对齐你想要的科技感与吸睛程度：

- **Windy（天气态势大屏标杆）**：层级清晰的图层系统 + 强动效粒子 + 高密度信息控制  
  - 链接：[`https://www.windy.com`](https://www.windy.com)
- **earth.nullschool（全球风场/气象可视化）**：极简 UI + 极强“科技感数据场”  
  - 链接：[`https://earth.nullschool.net`](https://earth.nullschool.net)
- **kepler.gl Demo（时间轴/刷选/空间过滤）**：时间滑块与地图联动的交互范式  
  - 链接：[`https://kepler.gl/demo`](https://kepler.gl/demo)
- **deck.gl Gallery（高性能图层与动效灵感）**：Heatmap/Trips/Arc 等动态图层表现  
  - 链接：[`https://deck.gl/gallery`](https://deck.gl/gallery)
- **ArcGIS Dashboards（地图+KPI+排名的成熟范式）**：指标、排名、地图联动“很企业但很稳”  
  - 链接：[`https://www.esri.com/en-us/arcgis/products/arcgis-dashboards/overview`](https://www.esri.com/en-us/arcgis/products/arcgis-dashboards/overview)
- **Map UI Patterns（地图交互模式库）**：Info Panel / Extent-driven content 等成熟模式  
  - 链接：[`https://mapuipatterns.com`](https://mapuipatterns.com)
- **Flightradar24（地图 + 列表/抽屉/详情的强交互）**：非常适合参考“排行榜反向驱动地图”的联动  
  - 链接：[`https://www.flightradar24.com`](https://www.flightradar24.com)
- **MarineTraffic（航运指挥中心风格）**：地图+态势+列表+详情抽屉  
  - 链接：[`https://www.marinetraffic.com`](https://www.marinetraffic.com)
- **Felt（协作 GIS + 图层+标注）**：现代 GIS 的产品感与图层交互  
  - 链接：[`https://felt.com`](https://felt.com)

## 7. 下一步（如果你要继续推进）

不细化实现的前提下，我建议下一轮只做 3 件事：

- **确定“主布局”**：A/B/C 选一个做主路线（我偏向 A）。
- **确定“省级态势 KPI 与排名字段”**：先列 8–12 个候选指标，避免后面 UI 反复推倒。
- **确定“右侧证据链的最小闭环”**：概览 KPI + 趋势（含风险标记）+ 关联散点 + 明细列表（可折叠）。
